# Hotel Booking System Presentation

---

## Project Overview
### A Java Console-Based Management System
- Focus: Room Reservations
- Audience: Hotel staff/managers
- Interface: Menu-driven

---

## Core Features
- **View Rooms**: List all available hotel rooms.
- **Room Details**: Display type, price, and booking status.
- **Booking**: Reserve a room for a customer.
- **Cancellation**: Cancel an existing reservation.
- **Error Management**: Robust handling of invalid inputs and unavailable rooms.

---

## Object-Oriented Principles Used
### Key Concepts:
1. **Abstraction**: Base `Room` class for shared logic.
2. **Inheritance**: `SingleRoom` and `DoubleRoom` subclasses.
3. **Polymorphism**: Dynamic method dispatch for `displayDetails()`.
4. **Encapsulation**: Private fields with getter/setter access.
5. **Exception Handling**: Custom logic for error scenarios.

---

## System Architecture
- **Room Class**: Abstract base for all room types.
- **Hotel Class**: Manages the room collection and booking logic.
- **Custom Exceptions**: Handlers for booking conflicts.
- **Main Class**: Entry point with the user menu.

---

## Why This System?
- **Efficient**: Simple console-based management.
- **Robust**: Strong OOP foundations and error handling.
- **Scalable**: Easy to add new room types or features.
